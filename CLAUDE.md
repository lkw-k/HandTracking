# Hand Tracking 프로젝트 개발 가이드

## 프로젝트 개요
웹캠으로 손을 인식하고 손가락 제스처를 통해 마우스를 제어하는 Python 애플리케이션입니다.

## 아키텍처

### 주요 구성
```
main.py - 단일 진입점
├── Hand Detection: MediaPipe Hands
├── Gesture Recognition: 손가락 위치/상태 기반 패턴 매칭
└── Mouse Control: pyAutogUI를 통한 자동화
```

### 핵심 변수
- `is_dragging`: 드래그 상태 플래그 (반드시 명시적 관리)
- `smoothing (0.23)`: 마우스 움직임 평활화 계수
- `landmark`: MediaPipe의 손 21개 포인트 (0-20)

## 코딩 규칙

### 손가락 제스처 추가 시
1. **is_finger_open()** 함수만 사용 (팁의 y좌표 < PIP의 y좌표)
2. **elif 체인 순서 중요**: 더 구체적인 조건을 먼저 배치
3. **제스처 충돌 방지**: 각 gesture는 상호배타적이어야 함
4. 새 제스처는 line 98-139 영역의 elif 블록에 추가

### 손 추적 성능 최적화
- 웹캠 해상도 (320x240) 변경 금지 - 정확도/성능 트레이드오프
- FPS 120 유지 (낮추면 제스처 반응성 저하)
- smoothing 값은 0.15~0.35 범위에서만 변경 (반응성 vs 안정성)

### 주석 작성
- WHY만 작성 (코드가 WHAT을 설명함)
- 손가락 landmark 번호와 의미는 주석 필수 (MediaPipe 공식 문서 참조)
- 제스처 로직의 의도만 명시

## 테스트 방침

### 수동 테스트 필수
- 각 제스처마다 최소 5회 반복 확인
- 다양한 조명 조건에서 테스트
- 손이 프레임 경계에 있을 때 동작 확인

### 테스트 체크리스트
- [ ] 마우스 이동 - 반응성, 부드러움
- [ ] 드래그 - 시작/진행/종료 상태 전환
- [ ] 클릭 (좌/우) - 오인식 없음
- [ ] 스크롤 - 방향, 속도 정확
- [ ] 5손가락 - 마우스 고정 작동
- [ ] 손 없음 → 드래그 해제 동작

## 보안 및 안정성

### 절대 금지
- `pyautogui.FAILSAFE` 비활성화 금지 (마우스 좌상단에 이동하면 중단)
- 하드코드된 절대 경로 사용 금지
- 웹캠 초기화 실패 시 예외처리 필수 (현재는 구현됨)

### 권장사항
- 드래그 중 예외 발생 시 `mouseUp()` 호출 보장
- 카메라 비활성화는 `cap.release()` 호출로 반드시 정리

## 의존성 관리
- mediapipe: 0.10.9 (고정) - 버전 변경 금지
- opencv-python: 최신 안정 버전
- pyautogui: 0.9.54 이상

`pyproject.toml`과 `uv.lock` 모두 동기화 필수.

## 디버깅 팁

### 제스처 인식이 안 될 때
1. `print()` 출력으로 손가락 상태 확인
2. `landmark` 좌표 값 확인 (0~1 정규화 범위)
3. 조명 조건 개선

### 마우스 움직임이 부자연스러울 때
- `smoothing` 값을 서서히 줄이기 (0.23 → 0.20)
- 웹캠 초점 조정

### 드래그가 풀리지 않을 때
- 카메라에서 손을 완전히 치우면 `mouseUp()` 호출됨
- 강제 리셋 필요시 프로그램 재시작 및 마우스 '마우스 클릭' 수동 해제
